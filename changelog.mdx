<Update label="March 27, 2026" description="Week of March 27">

### New features

**MCP endpoint** — Each docs site now exposes an MCP-compatible endpoint. AI tools that support the Model Context Protocol can connect to it to search your documentation content directly.

**Edit workflows** — Existing workflows can now be edited. Open a workflow and choose Edit to update its name, trigger, schedule, or connected repositories.

**Searchable repository pickers** — When creating or editing a workflow, the trigger repo and context repo fields are now searchable dropdowns. Type to filter your connected repositories instead of scrolling through a flat list.

**Trigger context in workflow runs** — The workflow runs list now shows where each run came from. PR-triggered runs link directly to the pull request on GitHub; scheduled runs show the time the run started.

### Bug fixes

- Failed workflow runs no longer surface internal error details in the run list. They now show a consistent message.

</Update>
